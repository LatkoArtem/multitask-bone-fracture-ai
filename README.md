## 1.1. Вибір та обґрунтування теми курсової роботи

**Тема:** Розробка інтелектуальної системи мультизадачної класифікації та семантичної сегментації переломів кісток на рентгенограмах.

**Обґрунтування вибору теми:**

Аналіз медичних зображень є ресурсомістким завданням для лікарів-рентгенологів. Втома та людський фактор призводять до діагностичних помилок, зокрема пропуску непомітних тріщин, що може мати критичні наслідки для здоров'я пацієнта.

Запропонований підхід базується на **двохетапному каскадному конвеєрі штучного інтелекту**, який:

1. **Етап скринінгу (класифікація):** мультизадачна модель одночасно визначає тип кістки та наявність перелому, відфільтровуючи здорові знімки.
2. **Етап деталізації (сегментація):** для підозрілих знімків точно локалізується область перелому шляхом генерації попіксельної маски.

Такий каскадний підхід має низку переваг:

- **Оптимізація обчислювальних ресурсів** — сегментаційна модель (більш ресурсомістка) запускається лише за потреби;
- **Підвищення інтерпретованості (Explainable AI)** — лікар бачить не лише ймовірність, а й підсвічену ділянку перелому на знімку;
- **Зниження кількості хибнонегативних результатів** — спільне навчання на кількох задачах покращує узагальнювальну здатність моделі.

## 1.2. Детальне формулювання прикладної задачі

### Мета

Створити інтелектуальну систему, здатну приймати рентгенівський знімок, паралельно визначати тип сфотографованої кістки та наявність перелому, і, у разі позитивного результату, генерувати просторову маску перелому.

### Простір станів ($S$)

Множина всіх можливих вхідних рентгенівських зображень, що формально описується як:

$$S = \{x \mid x \in \mathbb{R}^{C \times H \times W}\}$$

де:

- $C$ — кількість каналів зображення (1 для градацій сірого, 3 для RGB);
- $H, W$ — висота та ширина зображення у пікселях.

До простору станів також належать:

- Приховані стани — ознаки (feature maps), виділені згортковою нейромережею на проміжних шарах;
- Стани моделі — поточні ваги $\theta$ моделей під час навчання.

### Простір дій ($A$)

Система виконує наступну послідовність дій:

| #   | Дія                          | Формальний опис                                                                          |
| --- | ---------------------------- | ---------------------------------------------------------------------------------------- |
| 1   | Присвоєння мітки типу кістки | $a_1: x \mapsto \hat{y}_{bone} \in \{1, 2, \ldots, K\}$, де $K$ — кількість типів кісток |
| 2   | Оцінка ймовірності перелому  | $a_2: x \mapsto \hat{p} \in [0, 1]$                                                      |
| 3   | Активація модуля сегментації | $a_3: \text{activate if } \hat{p} > \tau$, де $\tau$ — порогове значення                 |
| 4   | Генерація попіксельної маски | $a_4: x \mapsto \hat{M} \in \{0, 1\}^{H \times W}$                                       |

### Метрики успішності

**Для класифікації:**

- **Sensitivity (Recall)** — максимізація виявлення справжніх переломів:

$$\text{Recall} = \frac{TP}{TP + FN}$$

- **Specificity** — правильне визначення нормальних знімків:

$$\text{Specificity} = \frac{TN}{TN + FP}$$

- **F1-score** — гармонічне середнє Precision та Recall:

$$F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$$

**Для сегментації:**

- **Коефіцієнт подібності Дайса (Dice Similarity Coefficient, DSC):**

$$\text{DSC} = \frac{2 |P \cap G|}{|P| + |G|}$$

де $P$ — передбачена маска, $G$ — ground truth маска.

- **Intersection over Union (IoU / Jaccard Index):**

$$\text{IoU} = \frac{|P \cap G|}{|P \cup G|}$$

## 1.3. Формалізація задачі (PEAS-опис системи)

Для формалізації задачі використаємо PEAS-фреймворк, що описує ключові характеристики інтелектуальної системи у медичному середовищі.

### PEAS-таблиця

| Компонент                                     | Опис                                                                                                                                                                                                                                                                                                                                                                 |
| --------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **P** (Performance measure — Міра успішності) | Точність постановки діагнозу (високий Recall ≥ 0.95); точність локалізації травми ($\text{IoU} \geq 0.7$); швидкість обробки одного знімка (інференс < 1 с); **мінімізація хибнонегативних результатів** (False Negatives) — найкритичніший показник у медичній діагностиці                                                                                          |
| **E** (Environment — Середовище)              | **Епізодичне** (кожен знімок обробляється незалежно), **повністю спостережуване** (система отримує весь знімок), **статичне** (знімок не змінюється під час обробки), **стохастичне** (різна якість знімків, шуми, артефакти), **дискретне** (кінцевий набір класів). Фізично — медична інформаційна система або датасет FracAtlas. Кількість систем — одна система. |
| **A** (Actuators — Механізми дії)             | Модуль генерації діагностичного звіту; візуалізатор (накладання маски перелому поверх оригінального рентгену); система сповіщення лікаря                                                                                                                                                                                                                             |
| **S** (Sensors — Сенсори)                     | Модуль прийому вхідних цифрових медичних зображень (у форматах JPG, PNG або DICOM)                                                                                                                                                                                                                                                                                   |

### Функція корисності системи

Система максимізує функцію корисності $U$, що формалізується як зважена сума метрик якості:

$$U(\theta) = \alpha \cdot \text{Recall}_{fracture} + \beta \cdot \text{F1}_{bone} + \gamma \cdot \text{DSC}_{seg} - \delta \cdot T_{inference}$$

де:

- $\alpha, \beta, \gamma, \delta$ — вагові коефіцієнти ($\alpha > \beta > \gamma > \delta$, оскільки пріоритетом є мінімізація пропуску переломів);
- $T_{inference}$ — час інференсу (штраф за повільну обробку);
- $\theta$ — параметри моделі.

## 2.1. Аналітичний огляд існуючих підходів

### 2.1.1. Класичні CNN для медичної класифікації

Архітектури глибоких згорткових нейронних мереж, такі як **ResNet** [1], **DenseNet** [2] та **ConvNeXt** [3], є стандартним інструментом для класифікації патологій на рентгенівських знімках. Ключова ідея ResNet — залишкові з'єднання (residual / skip connections), які вирішують проблему зникаючих градієнтів і дозволяють ефективно навчати дуже глибокі мережі (50–152 шари).

**Основний принцип ResNet** полягає у навчанні залишкової функції:

$$\mathcal{F}(x) = \mathcal{H}(x) - x \quad \Rightarrow \quad \mathcal{H}(x) = \mathcal{F}(x) + x$$

де $x$ — вхід блоку, $\mathcal{H}(x)$ — бажане перетворення.

**Недоліки однозадачного підходу:**

- Ігнорується додатковий контекст (наприклад, тип кістки може допомогти розрізнити перелом);
- Навчається лише на одному сигналі, що підвищує ризик перенавчання;
- Не забезпечує просторову локалізацію патології.

> **[1]** He K., Zhang X., Ren S., Sun J. _Deep Residual Learning for Image Recognition._ CVPR, 2016. DOI: 10.1109/CVPR.2016.90
>
> **[2]** Huang G., Liu Z., Van Der Maaten L., Weinberger K. Q. _Densely Connected Convolutional Networks._ CVPR, 2017. DOI: 10.1109/CVPR.2017.243
>
> **[3]** Liu Z., Mao H., Wu C.-Y. et al. _A ConvNet for the 2020s._ CVPR, 2022. arXiv:2201.03545

### 2.1.2. Мультизадачне навчання (Multi-task Learning)

**Multi-task Learning (MTL)** — це парадигма навчання, де модель одночасно вирішує кілька пов'язаних задач, використовуючи спільний екстрактор ознак (shared backbone) з кількома задачоспецифічними розгалуженнями (task-specific heads) [4].

**Переваги MTL у контексті медичної діагностики:**

1. **Регуляризація** — спільне вивчення кількох задач запобігає перенавчанню на одній із них;
2. **Ефективність представлення** — backbone навчається виділяти більш універсальні ознаки;
3. **Взаємна допомога задач** — знання типу кістки допомагає моделі краще розрізняти нормальний стан від перелому.

Загальна функція втрат MTL формулюється як:

$$\mathcal{L}_{total} = \sum_{t=1}^{T} \lambda_t \cdot \mathcal{L}_t$$

де $T$ — кількість задач, $\lambda_t$ — ваговий коефіцієнт $t$-ї задачі, $\mathcal{L}_t$ — функція втрат окремої задачі.

Для нашого випадку:

$$\mathcal{L}_{MTL} = \lambda_1 \cdot \mathcal{L}_{bone} + \lambda_2 \cdot \mathcal{L}_{fracture}$$

де:

- $\mathcal{L}_{bone}$ — Categorical Cross-Entropy для класифікації типу кістки:

$$\mathcal{L}_{bone} = -\sum_{k=1}^{K} y_k \log(\hat{y}_k)$$

- $\mathcal{L}_{fracture}$ — Binary Cross-Entropy для бінарної класифікації:

$$\mathcal{L}_{fracture} = -[y \log(\hat{p}) + (1 - y) \log(1 - \hat{p})]$$

> **[4]** Caruana R. _Multitask Learning._ Machine Learning, 28(1), 41–75, 1997. DOI: 10.1023/A:1007379606734

### 2.1.3. Архітектура U-Net для медичної сегментації

**U-Net** [5] є стандартом де-факто у медичній сегментації зображень. Архітектура має характерну U-подібну форму і складається з двох шляхів:

1. **Encoder (контрактивний шлях)** — поступово зменшує просторову роздільність, витягуючи семантичні ознаки;
2. **Decoder (розширювальний шлях)** — відновлює просторову роздільність, формуючи маску;
3. **Skip-connections** — передають просторову інформацію з encoder до decoder, що критично для точного обведення тонких ліній переломів.

![U-NET Architecture](u-net-architecture.png)

Функція втрат для сегментації — комбінована **Dice + BCE Loss**:

$$\mathcal{L}_{seg} = 1 - \text{DSC}(P, G) + \text{BCE}(P, G) = 1 - \frac{2 \sum_i p_i g_i}{\sum_i p_i + \sum_i g_i} - \frac{1}{N}\sum_{i=1}^{N}[g_i \log(p_i) + (1 - g_i)\log(1 - p_i)]$$

де $p_i$ — передбачена ймовірність для пікселя $i$, $g_i$ — ground truth значення.

> **[5]** Ronneberger O., Fischer P., Brox T. _U-Net: Convolutional Networks for Biomedical Image Segmentation._ MICCAI, 2015. arXiv:1505.04597

### 2.2.3. Опис основних класів та взаємодії

#### Клас `FracAtlasDataset(torch.utils.data.Dataset)`

**Модуль:** `data_module.py`

Відповідає за завантаження та передобробку даних з датасету FracAtlas.

| Метод / Атрибут                                                   | Опис                                                                                                                            |
| ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `__init__(root_dir, transforms, mode)`                            | Ініціалізація: шлях до даних, трансформації, режим (train/val/test)                                                             |
| `__getitem__(idx)` → `(tensor, label_bone, label_fracture, mask)` | Повертає тензор зображення $x \in \mathbb{R}^{3 \times 224 \times 224}$, мітку типу кістки, мітку перелому та маску сегментації |
| `__len__()` → `int`                                               | Повертає розмір датасету                                                                                                        |

#### Клас `MultiTaskClassifier(nn.Module)`

**Модуль:** `models.py`

Мультизадачний класифікатор із спільним backbone та двома task-specific heads.

| Метод / Атрибут                                 | Опис                                                                                                               |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `backbone`                                      | Попередньо навчана CNN (ResNet50, без останнього FC шару). Витягує вектор ознак $f \in \mathbb{R}^{2048}$          |
| `bone_head`                                     | `nn.Linear(2048, K)` — класифікація типу кістки                                                                    |
| `fracture_head`                                 | `nn.Linear(2048, 1)` — бінарна класифікація наявності перелому                                                     |
| `forward(x)` → `(logits_bone, logits_fracture)` | Прямий прохід: $x \xrightarrow{\text{backbone}} f \xrightarrow{\text{heads}} (\hat{y}_{bone}, \hat{p}_{fracture})$ |

**Формальний опис forward pass:**

$$f = \text{backbone}(x), \quad f \in \mathbb{R}^{2048}$$
$$\hat{y}_{bone} = W_{bone} \cdot f + b_{bone}, \quad \hat{y}_{bone} \in \mathbb{R}^{K}$$
$$\hat{p}_{fracture} = \sigma(W_{frac} \cdot f + b_{frac}), \quad \hat{p}_{fracture} \in [0, 1]$$

де $\sigma$ — сигмоїдна функція активації.

#### Клас `FractureSegmenter(nn.Module)`

**Модуль:** `models.py`

Реалізує архітектуру U-Net для семантичної сегментації переломів.

| Метод / Атрибут       | Опис                                                                                      |
| --------------------- | ----------------------------------------------------------------------------------------- |
| `encoder`             | Контрактивний шлях (4 блоки: Conv → Conv → MaxPool)                                       |
| `bottleneck`          | Центральний блок (Conv 1024)                                                              |
| `decoder`             | Розширювальний шлях (4 блоки: UpConv → Concat → Conv → Conv)                              |
| `forward(x)` → `mask` | $x \in \mathbb{R}^{3 \times H \times W} \mapsto \hat{M} \in [0, 1]^{1 \times H \times W}$ |

**Операція однієї енкодер-ступені:**

$$h_l = \text{ReLU}(\text{BN}(\text{Conv}_{3 \times 3}(\text{ReLU}(\text{BN}(\text{Conv}_{3 \times 3}(h_{l-1}))))))$$
$$h_l^{\downarrow} = \text{MaxPool}_{2 \times 2}(h_l)$$

**Операція однієї декодер-ступені:**

$$h_l^{\uparrow} = \text{UpConv}_{2 \times 2}(h_{l+1})$$
$$h_l' = \text{Concat}(h_l^{\uparrow}, h_l^{skip})$$
$$h_l^{out} = \text{Conv}_{3 \times 3}(\text{Conv}_{3 \times 3}(h_l'))$$

#### Клас `CascadePipeline`

**Модуль:** `inference.py`

Клас-обгортка, що реалізує каскадну логіку інференсу.

| Метод / Атрибут           | Опис                                            |
| ------------------------- | ----------------------------------------------- |
| `classifier`              | Завантажена модель `MultiTaskClassifier`        |
| `segmenter`               | Завантажена модель `FractureSegmenter`          |
| `threshold`               | Порогове значення $\tau$ (за замовчуванням 0.5) |
| `predict(image)` → `dict` | Повний каскадний конвеєр обробки знімка         |

### 2.2.4. Алгоритмічний опис каскадного конвеєра

**Алгоритм 1: Каскадний інференс (CascadePipeline.predict)**

```
АЛГОРИТМ: CascadeInference
ВХІД: зображення x ∈ ℝ^(C×H×W), поріг τ
ВИХІД: result = {bone_type, fracture_prob, mask (опціонально)}

1.  x_tensor ← preprocess(x)                    // Нормалізація, resize до 224×224
2.  f ← backbone(x_tensor)                      // Витяг ознак, f ∈ ℝ^2048
3.  logits_bone ← bone_head(f)                  // Логіти типу кістки
4.  logit_frac ← fracture_head(f)               // Логіт перелому
5.  ŷ_bone ← argmax(softmax(logits_bone))       // Передбачений тип кістки
6.  p̂ ← σ(logit_frac)                           // Ймовірність перелому
7.  result ← {bone_type: ŷ_bone, fracture_prob: p̂}
8.  ЯКЩО p̂ > τ:
9.      x_seg ← preprocess_for_segmentation(x)  // Resize до 512×512
10.     M̂ ← segmenter(x_seg)                    // Генерація маски
11.     M̂_binary ← (M̂ > 0.5)                    // Бінаризація маски
12.     result.mask ← M̂_binary
13. ІНАКШЕ:
14.     result.mask ← None                       // Перелом не виявлено
15. ПОВЕРНУТИ result
```

### 2.2.6. Математичний опис процесу навчання

#### Етап 1: Навчання мультизадачного класифікатора

Мінімізація зваженої суми функцій втрат:

$$\theta^* = \arg\min_{\theta} \frac{1}{N} \sum_{i=1}^{N} \left[ \lambda_1 \cdot \mathcal{L}_{CE}(\hat{y}_{bone}^{(i)}, y_{bone}^{(i)}) + \lambda_2 \cdot \mathcal{L}_{BCE}(\hat{p}^{(i)}, y_{frac}^{(i)}) \right]$$

де $N$ — розмір навчальної вибірки, $\theta$ — всі параметри моделі (backbone + heads).

**Оновлення ваг (Adam optimizer):**

$$m_t = \beta_1 m_{t-1} + (1 - \beta_1) \nabla_\theta \mathcal{L}$$
$$v_t = \beta_2 v_{t-1} + (1 - \beta_2) (\nabla_\theta \mathcal{L})^2$$
$$\theta_{t+1} = \theta_t - \eta \cdot \frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon}$$

де $\eta$ — learning rate, $\beta_1 = 0.9$, $\beta_2 = 0.999$.

#### Етап 2: Навчання сегментаційної моделі

$$\phi^* = \arg\min_{\phi} \frac{1}{|\mathcal{D}_{frac}|} \sum_{(x, M) \in \mathcal{D}_{frac}} \mathcal{L}_{seg}(\text{UNet}_\phi(x), M)$$

де $\mathcal{D}_{frac}$ — підмножина даних, що містять переломи, $\phi$ — параметри U-Net.

#### Повна цільова функція оптимізації

$$\min_{\theta, \phi} \quad \mathcal{L}_{total} = \underbrace{\lambda_1 \mathcal{L}_{bone} + \lambda_2 \mathcal{L}_{fracture}}_{\text{classification stage}} + \underbrace{\lambda_3 \mathcal{L}_{seg}}_{\text{segmentation stage}}$$

### 2.2.7. Структури даних та обмін інформацією

Взаємодія між компонентами відбувається через обмін багатовимірними масивами (**тензорами**):

| Тензор                 | Розмірність        | Опис                               |
| ---------------------- | ------------------ | ---------------------------------- |
| Вхідне зображення      | $(B, C, H, W)$     | Батч зображень, $B$ — розмір батчу |
| Feature map (backbone) | $(B, 2048)$        | Вектор ознак після GAP             |
| Logits bone            | $(B, K)$           | Логіти класифікації типу кістки    |
| Logit fracture         | $(B, 1)$           | Логіт наявності перелому           |
| Маска сегментації      | $(B, 1, H_s, W_s)$ | Попіксельна маска перелому         |
